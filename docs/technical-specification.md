# Техническое задание на разработку персонального сайта-портфолио

## 1. Введение

Данный документ описывает функциональные и нефункциональные требования к персональному сайту-портфолио IT-специалиста. Целью сайта является эффективное представление профессиональных навыков, опыта, проектов и личного бренда для потенциальных работодателей, клиентов и коллег.

## 2. Цели проекта

- Создание профессиональной онлайн-платформы для самопрезентации.
- Демонстрация выполненных проектов и компетенций.
- Ведение блога для публикации статей и обмена опытом.
- Предоставление удобного способа связи.
- Повышение узнаваемости и профессионального авторитета.

## 3. Функциональные требования

Сайт будет состоять из двух основных частей: **Главная страница** и **Блог**.

### 3.1. Главная страница (Single-Page)

Главная страница будет выполнена в формате "одностраничника" (single-page), состоящего из нескольких последовательных секций, на которые будут вести якорные ссылки из навигации.

- **Навигация:** Шапка сайта содержит ссылки (`/#about`, `/#projects`, `/#contact` и т.д.), которые обеспечивают плавную прокрутку к соответствующим секциям на этой же странице, а также ссылку (`/blog`) для перехода на отдельную страницу блога.

- **Секция 1: Приветствие**
    - Краткое приветствие и представление владельца сайта.
    - Основной слоган или направление деятельности.

- **Секция 2: Обо мне (About Me)**
    - Подробная биография и профессиональный путь.
    - Фотография владельца сайта.
    - Информация о мягких навыках (soft skills).

- **Секция 3: Компетенции (Skills/Competencies)**
    - Перечень технических навыков (языки программирования, фреймворки, инструменты, базы данных, методологии).
    - Описание областей экспертизы (например, автоматизация бизнес-процессов).

- **Секция 4: Опыт работы (Work Experience)**
    - Хронологический список мест работы.
    - Для каждой позиции: название компании, должность, период работы, основные обязанности и ключевые достижения.

- **Секция 5: Проекты (Portfolio)**
    - Перечень выполненных проектов.
    - Для каждого проекта: название, краткое описание, используемые технологии и ссылка на исходный код (GitHub) или демонстрацию.

- **Секция 6: Контакты (Contact)**
    - Электронная почта.
    - Ссылки на профили в социальных сетях и профессиональных платформах (LinkedIn, GitHub, Telegram).

### 3.2. Блог (Blog)

- **Отдельная страница**, на которую ведет ссылка из основной навигации.
- **Список статей:** На странице `/blog` отображается список всех статей с заголовками, датами публикации и краткими анонсами.
- **Детальная страница статьи:**
    - Открывается по адресу `/blog/[slug]`.
    - Содержит полный текст статьи, включая форматирование кода.
    - **Интеграция с YouTube:** Поддерживает встраивание видеороликов с YouTube непосредственно в текст статьи.

## 4. Нефункциональные требования

### 4.1. Производительность

- Быстрая загрузка страниц (Core Web Vitals - зеленые показатели).
- Оптимизация изображений и других медиа-файлов.

### 4.2. Адаптивность (Responsiveness)

- Сайт должен корректно отображаться и быть полностью функциональным на различных устройствах (десктопы, планшеты, смартфоны) и размерах экранов.
- Предпочтителен Mobile-first подход к дизайну.

### 4.3. Доступность (Accessibility)

- Соответствие базовым стандартам доступности (WCAG 2.1 AA) для обеспечения инклюзивности.

### 4.4. SEO (Search Engine Optimization)

- Использование семантической HTML-разметки.
- Наличие мета-тегов для каждой страницы (title, description).
- Генерация файла `sitemap.xml`.
- Поддержка разметки Open Graph для корректного отображения в социальных сетях.

### 4.5. Управление контентом

- Весь текстовый контент (статьи блога, описания проектов, информация "Обо мне") должен управляться через Markdown/MDX файлы, расположенные в репозитории проекта.

## 5. Технологический стек

- **Фреймворк:** Next.js (React)
- **Язык программирования:** TypeScript
- **Стилизация:** Tailwind CSS
- **Управление контентом:** Markdown/MDX
- **Развертывание (Deployment):** Vercel

- Аналитика посещений.
- Подписка на рассылку.
